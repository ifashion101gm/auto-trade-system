## 🎯 Project Overview

Automated cryptocurrency trading system featuring:

## 🏗️ Architecture Overview

### Core Components

#### 1. Execution Layer (`app/core/execution_engine.py`)
**Responsibility:** Order placement, lifecycle management, retry logic

**Key Features:**
- Thread-safe operations with asyncio locks
- Exponential backoff retry mechanism
- Order status tracking and reconciliation
- Latency optimization (<500ms target)

**Critical Concerns:**
- ⚠️ **Race conditions:** Use locks for concurrent order execution
- ⚠️ **Idempotency:** Prevent duplicate orders on retries
- ⚠️ **State consistency:** Ensure order state matches exchange state

**Common Issues:**
```python
# WRONG - No lock protection
async def place_order():
    await exchange.create_order(...)  # Race condition!

# CORRECT - With lock
async def place_order():
    async with self.order_lock:
        await exchange.create_order(...)
```


## 🔐 Safety Mechanisms

### 1. Daily Loss Limit
- **Threshold:** 2% of portfolio
- **Action:** Stop all trading, trigger circuit breaker
- **Reset:** Next trading day or manual override

### 2. Position Limits
- **Per position:** Max 5% portfolio value
- **Total exposure:** Max 20% portfolio value
- **Leverage:** Max 3x (currently using 1x)

### 3. Order Validation
- All orders validated before submission
- Quantity rounded to exchange step size
- Price checked against market price (±5%)
- Duplicate order detection

### 4. Circuit Breaker
- **Trigger:** 5 consecutive failures OR daily loss limit
- **Cooldown:** 30 minutes automatic
- **Reset:** Manual confirmation required

### 5. News Guard
- **Buffer:** 30 minutes before/after high-impact news
- **Sources:** Economic calendar API
- **Action:** Pause trading during events

### 6. WebSocket Reconnection
- **Max retries:** 10 attempts
- **Backoff:** Exponential (1s, 2s, 4s, 8s...)
- **Fallback:** REST API polling if WS fails


3. **Review risk validation:**
   ```bash

   ```


   ```bash

   ```

**Common Causes:**
- Rate limit exceeded → Check request frequency
- Insufficient balance → Verify account funds
- Invalid quantity → Check precision rounding
- Risk rejection → Review risk engine logs
- Network timeout → Check connectivity

**Time Estimate:** 30 minutes - 2 hours

### Task 4: Optimizing Performance

**Areas to Optimize:**

1. **Database Queries:**
   ```python
   # BAD - N+1 query problem
   trades = Trade.query.all()
   for trade in trades:
       print(trade.events)  # Separate query per trade
   
   # GOOD - Eager loading
   trades = Trade.query.options(joinedload(Trade.events)).all()
   ```

2. **Async Operations:**
   ```python
   # BAD - Sequential API calls
   balance = await exchange.fetch_balance()
   positions = await exchange.fetch_positions()
   
   # GOOD - Parallel API calls
   balance, positions = await asyncio.gather(
       exchange.fetch_balance(),
       exchange.fetch_positions()
   )
   ```

3. **Caching:**
   ```python
   # Cache instrument info (rarely changes)
   @cache(ttl=3600)  # 1 hour TTL
   async def get_instrument(symbol):
       return await exchange.fetch_market(symbol)
   ```

**Profiling Tools:**
```bash

python -m cProfile -o profile.stats scripts/test_cycle.py

# Memory profiling
pip install memory-profiler
python -m memory_profiler app/core/execution_engine.py
```

## 🚨 Emergency Procedures

### Scenario 1: Unexpected Losses

**Immediate Actions:**
1. Stop trading:
   ```bash
   curl -X POST http://localhost:8000/api/control/stop-trading
   ```
2. Check current positions:
   ```bash
   python scripts/check_positions.py
   ```
3. Review recent trades:
   ```bash
   psql -U postgres -d auto_trade -c "SELECT * FROM trades WHERE created_at > NOW() - INTERVAL '1 hour' ORDER BY created_at DESC;"
   ```
4. Assess damage and determine cause
5. Manual intervention if needed (close positions via exchange UI)

---

### Scenario 2: System Crash

**Recovery Steps:**
1. Check service status:
   ```bash
   systemctl status auto-trade-system
   journalctl -u auto-trade-system -n 50 --no-pager
   ```
2. Restart service:
   ```bash
   sudo systemctl restart auto-trade-system
   ```
3. Verify recovery:
   ```bash
   curl http://localhost:8000/health
   ```
4. Check for missed orders:
   ```bash
   python scripts/reconcile_orders.py
   ```
5. Review logs for root cause

---

### Scenario 3: Exchange API Down

**Fallback Strategy:**
1. Circuit breaker should auto-trigger
2. Switch to REST API polling if WebSocket down
3. Monitor exchange status page
4. Wait for recovery or manually close positions
5. Reconcile orders once API restored
